## What is `rke2-aws-tf`?

This is a **Terraform module** maintained by Rancher Federal that automates the deployment of **RKE2 (Rancher Kubernetes Engine 2)** clusters on AWS. Think of it as infrastructure-as-code that handles the heavy lifting of standing up a production-grade Kubernetes cluster in your AWS environment.

### What is RKE2?

RKE2 is Rancher's second-generation Kubernetes distribution, designed with a strong focus on **security and compliance** — particularly for government and federal use cases (hence "Rancher Federal" as the maintainer). It's hardened against CIS Kubernetes benchmarks out of the box and is notably lightweight compared to other enterprise Kubernetes distributions.

### What does this module actually provision?

The repo provisions RKE2 in a highly available, resilient, and scalable manner on AWS. Concretely, when you `terraform apply` this, you get:

- **AWS Auto Scaling Groups (ASGs)** — both server (control plane) nodes and agent (worker) nodes self-bootstrap and auto-join the cluster via userdata scripts, with no SSH required
- **An AWS Network Load Balancer (NLB)** — provides a static endpoint for the Kubernetes control plane
- **S3 bucket** — used as a secure secrets store for the cluster join token (instead of baking it into userdata)
- **IAM roles and policies** — minimum-privilege roles for nodes to fetch tokens, optionally upload kubeconfig, and support the AWS Cloud Controller Manager or Cluster Autoscaler

### The two modules you'll work with

The repo exposes two primary Terraform modules: the root `rke2` module which provisions the control plane load balancer and server nodepool, and the `agent-nodepool` module which creates worker ASGs that auto-join the cluster.

### Why it's designed the way it is

Since it's bad practice to store sensitive information in userdata, S3 is used as a secure secrets store for the cluster join token. Nodes boot, pull the token from S3 via the AWS CLI, and join the cluster — clean, auditable, and no secrets in plain text.

### Things to be aware of before you use it

A few things I'd flag from experience:

1. **Breaking changes in March 2023** — the `rke2` user is no longer installed by default, cloud-init scripts were renumbered, and the `asg` variable now requires a `termination_policy` value. If you're working from older examples or docs, watch for this.
2. **Requires Terraform >= 1.3 and AWS provider >= 5.32** — make sure your toolchain is current.
3. **It's a starting point, not a final solution** — the modules are intended as stepping stones for solutions that meet your workflow and organization's needs. You'll almost certainly need to extend it.
